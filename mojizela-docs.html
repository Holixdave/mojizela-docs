Quick Start

Get up and running

Clone the repo, configure your Firebase project, and you'll have Mojizela running on your device in under 10 minutes.

1

Clone and install dependencies

Clone the repository and run flutter pub get to install all packages.

2

Add your Firebase config

Download google-services.json (Android) and GoogleService-Info.plist (iOS) from your Firebase console and place them in the correct directories.

3

Configure Supabase

Update the Supabase URL and anon key in main.dart. Create the media storage bucket in your Supabase dashboard and set it to public.

4

Set up Agora for Live

Create an Agora project, copy your App ID into lib/core/services/agora_config.dart, and enable the RTC engine.

5

Run the app

Connect a device or emulator, then run flutter run. The native splash, intro screen, and auth flow will launch automatically.

Feed

Video feed

Mojizela's core loop — a vertical PageView of short videos and images, mixed with live cards, powered by a relevance-scored algorithm with shuffle.

🎬

For You Feed

Algorithm-ranked videos based on engagement score, freshness, and relevance. Infinitely paginates via loadMore().

👥

Following Feed

Videos + reposts from accounts the user follows. Friend reposts surface with a repost tag overlay.

🔴

Live Cards

Active live streams injected every 5 videos in the feed. Viewer count and host avatar ring displayed.

⚡

Pulse Bar

24-hour stories overlaid at the top of the feed. Photo, video, text, and voice note pulse types.

📤

Post Share Sheet

Share any post to private chats. Multi-select friends, confirm send, navigate to chat on success.

💡 The feed never shows an error message on load failure — instead it shows a branded empty-state SVG illustration with a retry button. No network, no ugly stack trace.

⚡

Pulse

Pulse stories

24-hour ephemeral content — photo, video, text cards, and voice note pulses. Each pulse expires automatically at creation time + 24h stored as a Firestore Timestamp.

⚠️ Pulses are NOT deleted from Firestore automatically — your Cloud Function or client-side filter on expiresAt must exclude expired entries. Always filter with .where('expiresAt', isGreaterThan: Timestamp.now()).
Dart · Creating a pulse 
final ref = _db.collection('pulses').doc();
await ref.set({
  'id':        ref.id,
  'userId':    uid,
  'mediaUrl':  mediaUrl,
  'mediaType': 'video',   // 'photo' | 'text' | 'audio'
  'caption':   caption,
  'createdAt': FieldValue.serverTimestamp(),
  'expiresAt': Timestamp.fromDate(
      DateTime.now().add(const Duration(hours: 24))),
  'viewedBy':  <String>[],
});
Live

Live streaming

Real-time audio/video broadcasting via Agora RTC. Hosts go live from the Live Lobby; viewers join via feed cards. Host detection drives the avatar ring UI.

🎙️

Agora RTC Engine

Low-latency audio/video using agora_rtc_engine ^6.5.3. Channel tokens generated server-side or via temp token for dev.

👁️

Viewer Count

Live viewer count stored on the Firestore lives doc and updated in real-time via RTDB presence counters.

💍

Avatar Ring

User avatar gets a pulsing gradient ring when they're live. Ring color matches the Mojizela brand gradient.

AI chat

Mojizela includes a built-in AI assistant powered by a Python backend at python-backend-ai.onrender.com. Conversation history is persisted locally via Hive.

Dart · AI service call 
final aiService = NewAIService(
  baseUrl: 'https://python-backend-ai.onrender.com',
);

// Send a message — returns a stream of tokens
await aiService.chat(
  messages: conversationHistory,
  onToken: (token) => setState(() => _buffer += token),
);
💡 The AI FAB is globally mounted in main.dart via an Overlay entry. It hides automatically when navigating to the AI chat screen and reappears on return.
Auth

Authentication flow

Mojizela uses Firebase Auth with email/password and Google Sign-In. A shadow ban check runs on every login to enforce community guidelines.

1

First launch → Intro screen

shared_preferences key intro_seen gates the intro. Once any CTA is tapped, it's marked seen and never shown again.

2

Auth choice → Login or Register

Users choose Google SSO or email. Register requires username + email + password. Username is indexed with search keywords on Firestore.

3

Email verification gate

Unverified email accounts are blocked from HomeScreen and redirected to EmailVerificationScreen with a resend option.

4

Shadow ban check on login

_ensureUserIntegrity() reads isBanned and isShadowBanned from the Firestore user doc. Banned accounts are signed out immediately.

FAQ

Frequently asked questions

Mojizela treats a failed feed load as a network issue — not a crash. A branded SVG illustration with a retry button is shown instead of a stack trace or generic error message. This keeps the UX polished and non-technical for end users.
If a user has no displayName or username in Firestore, AnonUsernameService.of(uid) generates a deterministic User<12digits> string by seeding dart:math Random with a hash of the uid. The same uid always produces the same number — no Firestore writes, fully device-side.
Yes — shadow banning sets isShadowBanned: true on the Firestore user doc. The _ensureUserIntegrity() check on login signs out the account and throws an exception with a "suspended" message. Regular bans use isBanned: true and show the same message.
Pulses are not auto-deleted. The expiresAt Timestamp is stored on the Firestore doc. All queries filter by .where('expiresAt', isGreaterThan: Timestamp.now()) to exclude expired entries. A Cloud Function can optionally clean them up server-side on a schedule.
The NewAIService wraps all requests in try/catch. If the backend at python-backend-ai.onrender.com is unreachable, an error message is shown in the chat bubble. The FAB is still visible and the user can retry once connectivity is restored.
